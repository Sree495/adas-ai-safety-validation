# Dataset: BDD-Sample (Berkeley Deep Drive)

This project uses a small subset of the BDD100K dataset in **Supervisely annotation format**.

The `data/` folder is excluded from this repository because:
- BDD100K is a UC Berkeley dataset with its own distribution licence
- Redistributing the raw images directly is discouraged by ML community convention

---

## How to obtain the data

### Option A — Download the BDD100K sample used in this project

The sample used here contains ~700 training images and ~109 validation images,
annotated in Supervisely format with 2 classes: `person` and `car`.

This is a custom-exported subset of BDD100K. To reproduce it:

1. Register at [https://bdd-data.berkeley.edu](https://bdd-data.berkeley.edu)
2. Download **BDD100K Images** (the 100K split) or the smaller **BDD100K MOT 2020** subset
3. Download **BDD100K Labels** (JSON format)
4. Export a ~700-image subset using the Supervisely platform or convert labels manually

### Option B — Use the Supervisely BDD export directly

If you have access to a Supervisely project with BDD annotations:

1. Export the project in **Supervisely JSON** format
2. Place it under `data/bdd-sample/` with this structure:

```
data/
└── bdd-sample/
    ├── train/
    │   ├── images/      ← JPEG images
    │   ├── ann/         ← per-image JSON annotations (Supervisely format)
    │   └── labels/      ← generated by src/preprocess.py
    ├── val/
    │   ├── images/
    │   ├── ann/
    │   └── labels/
    └── test/
        ├── images/
        └── ann/
```

---

## Annotation format

Each JSON file in `ann/` follows the Supervisely schema:

```json
{
  "size": {"height": 720, "width": 1280},
  "tags": [
    {"name": "weather", "value": "clear"},
    {"name": "timeofday", "value": "daytime"},
    {"name": "scene", "value": "highway"}
  ],
  "objects": [
    {
      "classTitle": "car",
      "points": {"exterior": [[x1, y1], [x2, y2]]}
    }
  ]
}
```

The `tags` array (weather / timeofday / scene) is used by `src/scenario_analysis.py`
to group images into evaluation scenarios.

---

## After placing data

Run the preprocessing step to generate YOLO-format label files:

```bash
python src/preprocess.py
```

Then verify the structure has `labels/` folders populated before training.
